# Memory Forest Agent Guide

This repository is a small Vite + React + TypeScript frontend prototype for exploring shared memories as connected timeline threads.

The app is currently client-side only:

- Data lives in React state.
- There is no backend or persistence layer.
- Refreshing the page resets the data.

## Repository Layout

### Top level

- `src/`: application source code.
- `guidelines/`: optional AI/project guidance notes. The current file is a template, not an active design system spec.
- `dist/`: production build output generated by Vite.
- `node_modules/`: installed dependencies.
- `index.html`: Vite HTML entry file.
- `package.json`: scripts and dependency manifest.
- `vite.config.ts`: Vite configuration.
- `postcss.config.mjs`: PostCSS configuration.
- `README.md`: product and usage overview for humans.
- `ATTRIBUTIONS.md`: third-party attribution notes.

### App entry points

- `src/main.tsx`: browser bootstrap; mounts the React app and imports global CSS.
- `src/app/App.tsx`: main application shell and the current state owner for people, events, dialogs, and selection state.

### App-specific code

- `src/app/components/`: feature components used by the prototype UI.
- `src/app/components/thread-canvas.tsx`: main visualization canvas for people, events, and their connections.
- `src/app/components/add-person-dialog.tsx`: create/edit flow for people.
- `src/app/components/add-event-dialog.tsx`: create/edit flow for events.
- `src/app/components/people-list.tsx`: sidebar list for people.
- `src/app/components/events-list.tsx`: sidebar list for events.
- `src/app/components/event-details-sheet.tsx`: event detail and edit entry point.
- `src/app/types/thread-memories.ts`: shared domain types such as `Person` and `Event`.

### Shared/generated UI layer

- `src/app/components/ui/`: reusable UI primitives in a shadcn/Radix style. Many of these appear to come from the original Figma Make export and are broader than the subset this app actively uses.
- `src/app/components/figma/`: small helpers from the export process.

Prefer changing app-specific components before editing `src/app/components/ui/` unless the change truly belongs in a shared primitive.

### Styling

- `src/styles/index.css`: global stylesheet entry.
- `src/styles/theme.css`: theme variables/tokens.
- `src/styles/tailwind.css`: Tailwind imports/layers.
- `src/styles/fonts.css`: font declarations.

## Working Notes

- The in-app label `ThreadMemories` is a temporary working name; the repository/project name is `Memory Forest`.
- The visualization is the heart of the prototype, so most behavioral changes will usually involve `src/app/App.tsx`, `src/app/components/thread-canvas.tsx`, or both.
- This repo does not currently include a test suite.
- Update this `AGENTS.md` if and only if you make a major structural repository change, such as moving core directories, adding new top-level app areas, or changing the main entry points.

## Common Commands

- `npm run dev`: start the local Vite dev server.
- `npm run build`: create a production build in `dist/`.
